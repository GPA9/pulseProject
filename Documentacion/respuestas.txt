# RESPUESTAS A LA DOCUMENTACIÓN

## 1. ROLES Y PERFILES
**¿Hay diferentes roles? ¿Músico o usuario, pero admin el quiero implementar, que será capaz de gestionar cualquier perfil?**
- Sí, hay 3 roles: `user`, `musician`, `admin`
- Admin ya está implementado y puede gestionar cualquier perfil
- El middleware AdminController protege las rutas admin

**¿Los músicos pueden editar perfil? Si**
- Sí, desde el panel de control pueden editar nombre, género, ciudad, bio, redes sociales y plataformas de streaming

**¿Puedes subir foto de perfil o banner?**
- Solo foto de perfil. Se sube a `images/band-logos/` y se muestra en el perfil del artista

## 2. MÚSICA
**¿Los músicos pueden subir canciones MP3 reales? Si**
- Sí, archivos MP3 reales se guardan en `storage/app/public/songs/`

**¿Las canciones se pueden reproducir desde la web? Si**
- Sí, con reproductor HTML5 `<audio>` en el perfil del artista

**¿Tienes reproductor global tipo Spotify?**
- No, es reproductor simple por canción individual

**¿Se registran reproducciones/escuchas a la BD?**
- Sí, cada vez que se reproduce una canción se incrementa `play_count` en la tabla `songs`

**¿Tienes likes, follows o playlists?**
- No implementados

## 3. RADIO

## ÚLTIMAS PREGUNTAS IMPORTANTES

### 1. SUBSCRIPCIONES
**¿Cómo se llaman los 3 planes de suscripción?**
- Basic (5€/mes)
- Pro (15€/mes) 
- Premium (30€/mes)

**¿Qué ofrece cada plan?**
- **Basic**: 5GB almacenamiento, hasta 20 canciones, conciertos básicos
- **Pro**: 15GB almacenamiento, hasta 100 canciones, merch avanzado, estadísticas
- **Premium**: 50GB almacenamiento, canciones ilimitadas, features premium, analytics avanzados

**¿Qué precio tiene cada plan?**
- Basic: 5€/mes
- Pro: 15€/mes
- Premium: 30€/mes

**Stripe: ¿utilizas Checkout Sessions, webhooks, Stripe Elements, o redirección simple?**
- Checkout Sessions para pagos
- Webhooks para actualizar estado de suscripción
- Redirección simple al checkout

**¿Cuando un usuario paga: se crea registro a la BD automáticamente?**
- Sí, en `SubscriptionController::success()` se crea registro con estado 'active'

**¿Se guarda estado de suscripción?**
- Sí, campo `status` (active, canceled, past_due, trialing)

**¿Se cancela automáticamente?**
- Sí, mediante webhook `handleSubscriptionDeleted`

### 3. BASE DE DATOS
**¿Relaciones importantes?**
- `User` → hasOne `MusicianProfile`
- `MusicianProfile` → hasMany `songs`, `concerts`, `merch`, `subscriptions`
- `Song` → belongsTo `MusicianProfile`, optional `Album`
- `Concert` → belongsTo `MusicianProfile`
- `Merch` → belongsTo `MusicianProfile`
- `Subscription` → belongsTo `MusicianProfile`

**¿Tienes claves foranes (foreign keys)?**
- Sí: `musician_profile_id`, `album_id`, `user_id`

**¿Tienes soft deletes o eliminación normal?**
- Eliminación normal con `delete()`

**¿Utilizas timestamps Laravel?**
- Sí, `created_at` y `updated_at` automáticos

### 4. SEGURETAT
